# statesman_scaffold — Installation Guide

## Requirements

- Ruby ≥ 3.2
- Rails 8.0+ (`activerecord`, `activesupport`, `railties`)
- `statesman` gem (peer dependency)

## Step 1: Add to Gemfile

```ruby
# Gemfile
gem "statesman"
gem "statesman_scaffold"
```

```bash
bundle install
```

## Step 2: Generate the Initializer

```bash
rails statesman_scaffold:install
```

This writes `config/initializers/statesman_scaffold.rb`:

```ruby
# frozen_string_literal: true

# Auto-include StatesmanScaffold::Concern into every ActiveRecord model
# so that the `with_state_machine` class macro is available application-wide.
#
# Generated by: rails statesman_scaffold:install
require "statesman"

Statesman.configure { storage_adapter(Statesman::Adapters::ActiveRecord) }

ActiveSupport.on_load(:active_record) do
  include StatesmanScaffold::Concern
end
```

Three responsibilities in one file:
1. `require "statesman"` — loads the gem before configuring it (avoids `NameError`)
2. Configures Statesman to use the ActiveRecord adapter
3. Auto-includes `StatesmanScaffold::Concern` into every AR model

**The task is idempotent:** running it again when the file already exists returns `:skipped`.

## Step 3: Generate State Machine Files for a Model

```bash
rails "statesman_scaffold:generate[Project]"
```

Creates:

```
app/models/project/state_machine.rb     # Project::StateMachine
app/models/project/transition.rb        # Project::Transition
db/migrate/TIMESTAMP_create_project_transitions.rb
```

Then run the migration:

```bash
rails db:migrate
```

## Step 4: Edit the StateMachine Class

Open `app/models/project/state_machine.rb` and define your states and transitions:

```ruby
class Project::StateMachine
  include Statesman::Machine

  state :pending, initial: true
  state :active
  state :done
  state :cancelled

  transition from: :pending,  to: %i[active cancelled]
  transition from: :active,   to: %i[done cancelled]

  after_transition do |model, transition|
    model.update!(status: transition.to_state)
  end
end
```

## Step 5: Add `with_state_machine` to the Model

```ruby
class Project < ApplicationRecord
  STATUSES = Project::StateMachine.states

  with_state_machine

  attribute :status, :string, default: Project::StateMachine.initial_state
  validates :status, inclusion: { in: STATUSES }, allow_nil: true
end
```

The `status` column is optional but recommended for fast reads without hitting the transitions table.

## Uninstalling

```bash
rails statesman_scaffold:uninstall
```

Removes `config/initializers/statesman_scaffold.rb`. Idempotent — safe when the file is already absent.

## Manual Include (without the initializer)

For non-standard setups or explicit control:

```ruby
class Project < ApplicationRecord
  include StatesmanScaffold::Concern
  with_state_machine
end
```

You must also configure Statesman manually:

```ruby
# config/initializers/statesman.rb
require "statesman"
Statesman.configure { storage_adapter(Statesman::Adapters::ActiveRecord) }
```

## Verifying the Installation

```ruby
# Rails console
StatesmanScaffold::Concern              # should not raise NameError
ActiveRecord::Base.ancestors.include?(StatesmanScaffold::Concern)  # => true

project = Project.create!(name: "Test")
project.current_state                   # => "pending" (or your initial state)
project.in_state?(:pending)             # => true
```

## Troubleshooting

**`NameError: uninitialized constant Project::StateMachine`**
→ Run `rails "statesman_scaffold:generate[Project]"` before adding `with_state_machine` to the model.

**`ActiveRecord::StatementInvalid: no such table: project_transitions`**
→ Run `rails db:migrate` after generating files.

**`NameError: uninitialized constant Statesman`**
→ Re-run `rails statesman_scaffold:install` to get the initializer with `require "statesman"`.

**`Statesman::TransitionFailedError`**
→ Transition is not defined for the current state. Check `transition from:` rules in `StateMachine`.

**Status column out of sync with `current_state`**
→ Add `after_transition { |m, t| m.update!(status: t.to_state) }` to `StateMachine`.

**Generator creates files but `with_state_machine` still raises**
→ Ensure the generated files are in Rails' autoload paths (`app/models/project/`). Run `Rails.autoloaders.log!` to debug.
